# Public / Private Boundary

Public repositories contain framework material, schemas, templates, examples and sanitised references.

Private nodes contain live operating data, secrets, CRM details, payment data, private business strategy, private partner context and deployment specifics.

## Public-Safe

Public material may include:

- protocol descriptions
- organism architecture
- schema definitions
- Light Node card templates
- generic source-of-truth patterns
- sanitised examples
- validation badge definitions
- public claims boundaries
- generic recovery patterns
- commons governance rules

## Private-Only

Private material includes:

- API keys
- customer data
- private emails
- payment exports
- CRM exports
- individual user records
- internal financial detail
- private client strategy notes
- sensitive partner information
- security-sensitive deployment details
- non-redacted outcomes
- model/provider secrets

## Network Sharing Rule

Kastel network learning should share redacted patterns, not raw private truth.

A shared artefact should include:

- context
- validation badge
- known limits
- failure modes
- adaptation notes
- source-of-truth category
- privacy / shareability level

It should not include customer data, private financials, partner secrets or anything that would compromise the local node.
