# Source of Truth Registry

Every important business object should identify one primary owner system.

Other systems may mirror, cache, summarise or enrich facts, but they should not silently become authoritative.

In the adaptive organism model, the source-of-truth registry protects sensory integrity. A business cannot learn cleanly if its sensors are ambiguous or its memory banks false signals.

## Registry Questions

For each object, define:

- object name
- object type
- primary source-of-truth system
- allowed mirrors or exports
- update frequency
- owner
- privacy level
- recovery route
- validation use
- whether it can be shared with the network commons

## Common Source-of-Truth Objects

- customer record
- purchase
- subscription
- refund
- entitlement
- product session
- completion
- support issue
- claim boundary
- proof object
- partner contact
- content page
- search query
- email event

- Clarify Intent
- delayed review
- script validation status
- recovery protocol

## Anti-Capture Rule

No external platform should become the only place where critical business truth exists.

Each critical source should have at least one of:

- export
- local backup
- secondary mirror
- manual fallback
- recovery procedure
- documented replacement route

This is part of Kastel's shark-tooth continuity posture.
